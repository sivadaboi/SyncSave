# SyncSave Privacy Policy

**Effective Date: May 24, 2026**

SyncSave is a fully open-source, local-first, peer-to-peer (P2P) game save synchronization utility. Because we believe that privacy is a fundamental right, SyncSave is designed with **data minimization and user custody** at its core. 

---

## 1. Zero Personal Data Collection
- **No Accounts:** SyncSave does not require registration, email addresses, usernames, or passwords.
- **No Telemetry or Analytics:** We do not collect crash reports, usage statistics, device tracking details, or telemetry of any kind. 
- **No Tracking Cookies:** The dashboard runs locally on your machine and uses no analytics cookies or trackers.

## 2. Local-First Storage & Custody
- **Local Database:** All records of monitored games, save locations, paired peers, and settings are stored locally on your machine in a JSON database (`.syncsave/db.json` or equivalent system folder).
- **Save Backups:** All save snapshots and branch histories are zip-compressed and stored strictly on your local hard drive (in the configured backups directory). No saves are ever uploaded to a proprietary cloud service.

## 3. Peer-to-Peer Data Transmission
SyncSave replicates your saves directly between your paired devices using two connection methods:

### A. Local Area Network (LAN) Sync
- **UDP Discovery:** On your home network, SyncSave broadcasts a lightweight heartbeat (containing only your device name) to discover other local SyncSave instances.
- **Direct P2P:** File synchronization occurs directly between your devices over local TCP sockets. No internet connection is required for LAN syncing.

### B. Wide Area Network (WAN) Relay Sync
- **Firewall Traversal:** To sync saves over the Internet (e.g. between different cities or NAT firewalls), both peers connect outbound to a configured WebSockets Relay Server URL.
- **Matchmaking Room:** Devices match using a temporary "Sync Room Code" generated by the users.
- **No File Logging:** The relay server acts strictly as a traffic transit tunnel (NAT traversal) to forward encrypted binary blocks. The relay server **does not log, store, or inspect** your files. Once a file chunk is forwarded to the destination peer, it is immediately discarded from the relay server's active memory.
- **Self-Hosting:** If you do not trust a public relay server, SyncSave allows you to host your own local relay server directly from the settings panel.

## 4. Open-Source Transparency & Auditing
Because SyncSave is 100% open-source:
- You are free to inspect the source code to verify exactly how your files are read, packaged, and transmitted.
- You can build the application from source code directly to ensure that no hidden scripts or modifications have been made.

## 5. Third-Party Links & Integrations
- **Steam Store API:** During local scans, SyncSave may query the public Steam API (`store.steampowered.com`) using Steam AppIDs to automatically resolve game folder numbers (like `480` or `1245620`) to real game names. These queries are anonymous, contain no personal identifiers, and do not track your user profile.

## 6. Updates & Inquiries
As an open-source project, any future changes to this policy will be committed directly to the project repository for public review. If you have questions or want to review the code, please visit the repository or contact the developer.
